RWD Retrofitting an Older Enterprise Site

Patrick Boggs - Interface Developer at Manheim.com

Corporate Web World

  • Code is old
  • Decisions are hard
  • Too many people involved

It’s too hard...

Don’t say “we need to be responsive”

“Let’s make the site work on any screen size,
We can do it mostly with CSS”

Work with what you have

  • CSS Zen Garden
  • Flexible units. Pixels are dead

Compromise

UI/UX, Product and Visual design

Do what works for the customer

Iterate

Make the retrofit a starting point, not the final product

Use a little JavaScript

You can remove classes and IDs...

Then add more tech if required and possible

The Hard parts...

  • Tables/Data Grids
  • Media (other than images)
  • Modals, Accordions and the like
  • Tables/Data Grids

Convincing stakeholders

  • Roll up your sleeves, prove it works
  • Work WITH product owners SHOW them what you can do with CSS
  • Make sure that is an ongoing process
  • Tables/Data Grids are Really a pain

In the future

  • Like Fluid websites, RWD is a baseline from here on out
  • Code everything in small modular parts
  • Design them ahead of time to be fluid and responsive
  • Make screen sizes part of your workflow
